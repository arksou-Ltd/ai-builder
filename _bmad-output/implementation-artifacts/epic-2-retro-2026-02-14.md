- Review 返工密度偏高：Story 2.3/2.4 出现多轮高优修复，暴露“开发完成 ≠ 可评审完成”。
- 状态治理存在漂移：`sprint-status.yaml` 中 `epic-2` 仍为 `in-progress`，与故事完成现状不一致。
- 测试策略曾反复：testcontainers、真实环境、依赖注入边界在不同轮次出现口径不一致。
- 文档与代码对齐成本较高：File List、迁移说明、状态说明多次补齐。

## Story Pattern Synthesis
- Common Struggles:
  - 契约与边界条件（401/404/422、参数合法性、失败可恢复）是重复出现最多的问题域。
  - 数据模型与迁移一致性（字段语义、nullable 约束、downgrade 对齐）需要更早锁定。
- Recurring Review Feedback:
  - 高优问题主要集中在“语义正确性”和“测试证据真实性”，而非纯代码风格。
  - 中优问题集中在“用户错误感知”和“状态污染/副作用控制”。
- Breakthrough Moments:
  - 登录与 API 鉴权路径统一后，后续 Story 的身份与访问控制问题显著减少。
  - 删除失败回滚与重试路径补齐后，前端可靠性交互标准得到固化。

## Significant Discoveries and Plan Impact
### Discovery 1: State Reconciliation Gap
- 现象：故事已全部完成，但 `epic-2` 状态未自动收敛到完成态。
- 影响：若不修复，Epic 3 的状态面板可能继续展示“看似进行中、实际已完成”的错误信号。
- 建议：在 Epic 3 增加“Epic/Story 状态一致性校验与自动收敛”能力。

### Discovery 2: Quality Gate Positioning Is Too Late
- 现象：高优问题多数在 Review 后期暴露，导致多轮返工。
- 影响：影响吞吐稳定性和交付可预测性。
- 建议：在 Epic 3 的工作流步骤中加入“进入 Review 前质量门槛检查”与“证据完整性检查”。

### Discovery 3: Testing Policy Needs Explicit Workflow Enforcement
- 现象：No Mock 约束虽最终执行，但中途有偏离与修正。
- 影响：测试可信度依赖人工纠偏。
- 建议：把 No Mock 规则纳入步骤面板可见检查项与阻断条件。
